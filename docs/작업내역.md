### 1. 인증 시스템 (자체 구현, 라이브러리 무의존)

Node.js 내장 `crypto`만으로 세션 기반 인증을 구현. 서버 세션 + HMAC 서명 쿠키 방식(JWT 아님 → 로그아웃/권한변경 즉시 반영).

| 파일 | 역할 | 핵심 |
|---|---|---|
| `src/lib/password.ts` (신규) | 비밀번호 해싱/검증 | `scrypt`(N=16384,r=8,p=1). 포맷 `scrypt$N$r$p$salt$hash`. `DUMMY_HASH`로 사용자 존재 유추 방어, `timingSafeEqual` 비교 |
| `src/lib/session-crypto.ts` (신규) | 세션 쿠키 서명 | `HMAC-SHA256`, 시크릿 `AUTH_SECRET`. 쿠키명 `pms_session`, 포맷 `<sessionId>.<hmac>`. DB/next-headers 무의존(proxy에서 재사용) |
| `src/server/session.ts` (신규) | 세션 생명주기 | `createSession`(TTL 7일, httpOnly·sameSite=lax·prod에서 secure), `getSessionUser`(쿠키검증→DB조회→만료검사), `destroySession` |
| `src/server/auth.ts` (수정) | 인증 진입점/가드 | `getCurrentUser`, `getScope`(현재 `where:{}` — 향후 테넌트 격리 seam), `requireUser`(→`/login`), `requireAdmin`(비관리자→`/`) |
| `src/proxy.ts` (신규) | Next 16 미들웨어 | 요청 쿠키의 HMAC만 검증(DB 접근X). 무효 시 `/login` 리다이렉트. matcher에서 `/login`·`/api`·`_next/**` 제외 |
| `src/app/login/page.tsx`, `src/components/auth/LoginForm.tsx` (신규) | 로그인 화면 | `login` 액션 호출 |

- **env:** `AUTH_SECRET`(16자+ HMAC 시크릿) 필요.

**핵심 로직(서버 액션):**
- `createUser`: ADMIN 전용, 이메일 중복 검사, 관리자 workspace 상속.
- `updateUser`: 이메일 변경 시에만 중복 재검사 + P2002 방어. **본인 관리자 권한 해제 금지**, **마지막 관리자 강등 금지**. 비밀번호 공란이면 기존 유지.
- `deleteUser`: **본인 삭제 금지**, **마지막 관리자 삭제 금지**. 트랜잭션으로 `note.authorId`/`project.ownerId`/`task.assigneeId`를 NULL 처리하고 세션 삭제 후 사용자 삭제 → DB의 FK 설정과 무관하게 안전.

**이번 세션에서 추가한 부분:** `updateUser`/`deleteUser` 액션, `updateUserSchema`, UserAdmin의 수정 모달·삭제 UI·본인 행 가드, `page.tsx`의 `currentUserId` 전달. (생성/로그인은 이전 작업.)

### 3. 나라장터 입찰공고 수집 (G2B)

공공데이터포털 나라장터 API로 **용역** 입찰공고를 회사 키워드(AI·AX·데이터·챗봇)로 수집·선별·관리. 설계·계획 문서는 `docs/superpowers/{specs,plans}/2026-07-14-g2b-*` 참고.

| 파일 | 역할 | 핵심 |
|---|---|---|
| `src/lib/g2b.ts` (신규) | 키워드 그룹 SSOT | `BID_KEYWORD_GROUPS`(ai/ax/data/chatbot), `matchKeywordGroups`, `termsForGroups` |
| `src/server/g2b/client.ts` (신규) | API 클라이언트(`server-only`) | `getBidPblancListInfoServcPPSSrch` 호출, 단일/배열 응답 파싱, `isG2bAvailable()` |
| `src/server/g2b/collect.ts` (신규) | 수집/정규화(`server-only`) | 검색어별 페이지 순회(100건), `(bidNtceNo,bidNtceOrd)` dedup, KST↔UTC 변환 |
| `src/server/actions/bids.ts` (신규) | 서버 액션 | `collectBids`(upsert, 사용자 status/memo 보존), `setBidStatus` |
| `src/app/(app)/bids/page.tsx`, `src/components/bids/BidWorkbench.tsx` (신규) | 입찰 화면 | 진행중/관심 탭, 검색·정렬, 관심(하트) 토글, 수집 컨트롤 |

- **DB:** `BidNotice` 모델 + `BidStatus`(NEW/INTERESTED/EXCLUDED) enum, `@@unique([bidNtceNo, bidNtceOrd])`.
- **env:** `G2B_SERVICE_KEY`(선택, 미설정 시 수집 버튼 비활성).
